# Home Assistant Stroomprijsprognose Integration

Custom Home Assistant component for electricity price forecasts from [stroomprijsprognose.nl](https://stroomprijsprognose.nl). Provides hourly retail electricity prices (incl. grid fees, levies, VAT) based on your postal code.

> **AI-Assisted Development**
> This integration was initially generated by AI and has been iteratively reviewed, tested, and improved. 88 automated tests cover coordinator logic, sensor values, config flow validation, and caching. Use responsibly for financial decisions.

## Features

- 72-hour electricity price forecast via [stroomprijsprognose.nl API](https://stroomprijsprognose.nl/api/v1/hourly-forecast)
- Retail prices include: grid fees, levies, taxes, VAT — automatically determined by postal code
- 15 sensors: current/next hour price, min/max/avg, price level classification, best times, diagnostics
- Smart API caching — derived values recomputed locally between fetches
- Configurable via Home Assistant UI (no YAML needed)
- Services: force refresh (bypass cache), get prices (returns data for templates)
- Config flow validates API connectivity before creating entry
- Reconfigure postal code/country without removing the integration
- Translations: English, German, Dutch, French

## Installation

### HACS (recommended)

1. Add this repository as custom repository in HACS
2. Install "Stroomprijsprognose Electricity Prices"
3. Restart Home Assistant

### Manual

```bash
cd /path/to/your/homeassistant/config
cp -r custom_components/stroomprijsprognose custom_components/
# Restart Home Assistant
```

## Setup

1. Go to **Settings → Devices & Services → Add Integration**
2. Search for "Stroomprijsprognose"
3. Enter your postal code and select your country
4. The integration validates API connectivity before creating the entry
5. Click Submit

The integration fetches data every 15 minutes (configurable 5–60 min). Between fetches, time-dependent values are recomputed locally from cached data. Adjust interval in the integration options.

### Reconfigure

Change your postal code or country without removing the integration:

1. Go to **Settings → Devices & Services**
2. Click **Configure** on the integration entry
3. Click the three-dot menu → **Reconfigure**

### Services

**Force refresh** — bypass cache and immediately re-fetch from API:

```yaml
# Refresh all instances
service: stroomprijsprognose.force_refresh

# Refresh a specific instance
service: stroomprijsprognose.force_refresh
data:
  entry_id: "abc123..."
```

**Get prices** — return cached price data as a service response:

```yaml
# Access prices in templates
service: stroomprijsprognose.get_prices
response_variable: prices
```

## Sensors

| Sensor | Unit | Default | Category | Description |
|--------|------|---------|----------|-------------|
| `current_price` | ct/kWh | enabled | primary | Retail price for current hour |
| `next_price` | ct/kWh | enabled | primary | Retail price for next hour |
| `lowest_price` | ct/kWh | enabled | primary | Cheapest retail price in forecast |
| `highest_price` | ct/kWh | enabled | primary | Most expensive retail price |
| `average_price` | ct/kWh | enabled | primary | Average retail price over forecast |
| `price_level` | — | enabled | primary | Price classification: very_cheap to very_expensive |
| `lowest_price_time` | datetime | enabled | primary | When the cheapest hour is |
| `highest_price_time` | datetime | enabled | primary | When the most expensive hour is |
| `lowest_next_8h_price_time` | datetime | enabled | primary | Cheapest slot in next 8 hours |
| `price_percentage` | % | disabled | primary | Current price as % of forecast maximum |
| `current_forecast_price` | ct/kWh | disabled | diagnostic | Model forecast price for current hour |
| `current_day_ahead_price` | ct/kWh | disabled | diagnostic | Day-ahead auction price for current hour |
| `price_source` | — | disabled | diagnostic | Price origin: `day_ahead` or `forecast` |
| `forecast_slots` | h | disabled | diagnostic | Count of model-forecast slots in window |
| `last_updated` | datetime | disabled | diagnostic | When API data was last fetched |

See [docs/sensors.md](docs/sensors.md) for full sensor reference.

## Automation Examples

Start washing machine when price is cheap or very cheap:

```yaml
alias: "Washing machine on cheap power"
trigger:
  - platform: state
    entity_id: sensor.stroomprijsprognose_price_level
    to:
      - "very_cheap"
      - "cheap"
action:
  - service: switch.turn_on
    target:
      entity_id: switch.washing_machine
```

Start dishwasher when next hour is cheaper:

```yaml
alias: "Dishwasher when next hour is cheaper"
trigger:
  - platform: template
    value_template: >
      {{ states('sensor.stroomprijsprognose_next_price') | float <
         states('sensor.stroomprijsprognose_current_price') | float }}
action:
  - service: switch.turn_on
    target:
      entity_id: switch.dishwasher
```

More examples: [docs/automations.md](docs/automations.md)

## Template Usage

Access forecast data in templates:

```jinja2
{# Cheapest hour today #}
{{ state_attr('sensor.stroomprijsprognose_current_price', 'cheapest_slots')[0].timestamp }}

{# Price in 3 hours #}
{{ state_attr('sensor.stroomprijsprognose_current_price', 'hourly_prices')[3].retail_total_ct_kwh }}

{# Current price as percentage of max #}
{{ states('sensor.stroomprijsprognose_price_percentage') }}%

{# Is now a good time? #}
{{ states('sensor.stroomprijsprognose_price_level') }}
```

## API

This integration uses the free public API at [stroomprijsprognose.nl](https://stroomprijsprognose.nl).

- Endpoint: `/api/v1/hourly-forecast`
- Timeout: 30 seconds
- Rate limiting: Unknown, be reasonable
- No API key required
- Data updates roughly every 15 minutes
- Smart caching reduces API calls by ~50% (derived values recomputed locally)
